Since the last blog entry, I have gone deeper into depth of what nvm and npm really do. We can use these commands in your command line to do different things, depending on whether you start with the nvm or the npm command.

### Takeaways(nvm)
To help understand this, the command nvm stands for Node Version Manager, so most nvm commands are used to switch between version controls, and to download versions. We use nvm because we already downloaded node.js, and for us to work with different projects, we'll need different versions. If we aren't on the right version, our package won't work, or it might even just break. We can also use nvm to test different node versions to make sure they don't crash. Basically, nvm is how we switch between versions, and how we check if node packages crash on a specific version.

### Takeaways(npm)
The `npm` command stands for Node Package Manager. We can use the npm command to create the json package that we can use later on with ```express.js```. This is probably the most important part of node.js as a whole, since it's the main thing used to share code, and without it, you'd probably spend a long time on writing out code that could be accessed with shortcuts through npm. We also want to use npm to keep the same node version for our project, as changing versions as stated earlier, would likely cause the project to break. With this being said, the npm is the most important of the two commands in my opinion, because it's what gets the actual code running.
